How HTML, CSS, and JavaScript work together.

When I first started learning how websites work, I needed a simple way to understand the relationship between HTML, CSS, and JavaScript.

Think of building a website like building a house.

— HTML is the structure (walls, doors).

— CSS is the style (paint, furniture).

— JavaScript is the behavior (lights, movement).

HTML (the structure)

This gives us a title, some text, and a button. But it’s plain and doesn’t do anything. This is the foundation. Without it, there's nothing to look at. 


      <h1>Welcome to my website</h1>
      <p>This is a short description.</p>
      <button>Click me</button>

CSS (the style)

Now we paint the walls and hang up curtains.


h1 {
  color: darkblue;
}

button {
  background-color: coral;
  border-radius: 8px;
  padding: 10px;
}

Now the heading is blue and the button looks nice.

JavaScript (the interaction)

JavaScript is what makes the house “smart.” We press the button and something happens. 


    document.querySelector("button").addEventListener("click", function() {
      alert("You clicked the button!");
    });

Now when someone clicks the button a pop-up message appears. JavaScript made our page interactive.

What is Control Flow and Loops.

Think of control flow as the step-by-step instructions to complete a task. Like making a cup of tea.

Step-by-Step Instructions


1. Boil water
2. Put a tea bag in the cup
3. Pour hot water into the cup
4. Wait until it’s done brewing
5. Take the tea bag out
6. Add milk or sugar if you want
7. Drink the tea

This is a step-by-step flow. The same as how code runs line by line.

Loop Example

We check if the kettle is done boiling. If not, we keep checking.


while (kettleIsNotBoiled) {
  checkKettle();
}

You’re repeating the check until the kettle boils. That’s a loop. Keep doing something while a condition is true.

If Statement Example

This is a condition. Only add milk if the person wants it.


// JavaScript code
if (wantsMilk) {
  addMilk();
}

Summary

  • Control flow = steps followed one by one
  • If statement = do something only if a condition is true
  • Loop = keep doing something until a condition changes

What is the DOM?

DOM stands for Document Object Model. It’s like a map of the web page that browser creates when it loads an HTML file.

JavaScript uses the DOM to see and change what's on the page. Like updating text, hiding elements, or changing styles.

Example of interacting with the DOM

Let’s say we have this HTML

<p id="message">Hello!</p>
    <button>Change Message</button>

And this JavaScript

document.querySelector("button").addEventListener("click", function() {
      document.getElementById("message").textContent = "You clicked the button!";
    });

When we click the button, JavaScript finds the paragraph (using the DOM), and changes its text. That’s interacting with the DOM.

Overall, without the DOM, our page would just sit there and never respond to anything. No clicks, no updates, nothing.

Difference between accessing data from Arrays and Objects

Arrays

An array is an ordered list of items. Each item has a number (called an index), starting from 0. 

let fruits = ["apple", "banana", "cherry"];

To access the second item in the array we write code:

console.log(fruits[1]);

The output will be "banana"

Arrays are useful when we need to store a list and care about the order of the items.

Objects

An object stores key–value pairs. We access the data using the key (also called a property name), not a number. 

let person = {
    name: "John",
    age: 35,
    job: "sailor"
  };

To access the person’s job:

console.log(person["job"]);

Objects are useful when each piece of data has a label or identifier.

What are Functions and why are they helpful?

A function is a reusable block of code that performs a specific task. We can think of it like a mini-program inside a bigger program.
Instead of writing the same code again and again, we can write it once inside a function, and then run (or “call”) that function whenever we need it.

Functions are useful because they help to write cleaner, more efficient, and more organized code.

Example of a Function

This function says hello


    function greet() {
      console.log("Hello!");
    }

    greet();

Function with a Parameter

This function greets a person by name


    function greet(name) {
      console.log("Hello, " + name + "!");
    }

    greet("Anna"); 
    greet("Leo");

In this example, name is a parameter.It lets the function work with different inputs.

Functions help to avoid repeating code by letting reuse the same block whenever needed.
